# Phase 0: Orientation and Learning Environment

**Duration:** 2 weeks
**Purpose:** Understand the journey and get a working development environment.
**Needs:** No API keys. No cloud account required by the learner (a laptop is enough).

## CBUSI contents

- **C - Concepts:** [`concepts/01-understanding-the-ai-profession.md`](concepts/01-understanding-the-ai-profession.md) - the AI vocabulary, grounded and cited.
- **B - BUILD:** environment setup (Python, git, GitHub, Docker, PostgreSQL, venv, pip) + create the `ai-consulting-learning-journey` portfolio repo. *(tested + output captured)*
- **U - USE:** stand up the study and portfolio system (journal, glossary, templates, experiment log).
- **S - SURVIVE:** broken virtual environment, git disaster (committed secret / detached HEAD), Docker port clash.
- **I - Interview:** MLOps vs LLMOps, AI vs ML vs generative AI, keeping work reproducible.

## Exit standard

Your environment works end to end: you can create a repo, commit, push, run a Python script inside a virtual environment, and run a container.

## Proof of competence

The `ai-consulting-learning-journey` repo exists on GitHub with all five Phase 0 deliverables committed (learning goals, study calendar, technology inventory, AI glossary, portfolio roadmap), and `docker run` executes a hello-world Python container on your machine.
